# 递归解析 (Recursive Descent Parsing)：让机器拥有“套娃”能力

要让计算机看懂 `1 + ( 2 * (3 - 1) )` 这种可以无限往里套括号的算式，大牛们发明了一种代码写法，叫做 **递归下降解析 (Recursive Descent)**。

什么叫“下降”？什么叫“递归”？我们来看几段最核心的伪代码逻辑。

## 1. 设定三个职级分明的“办事员”（解析函数）

在 Parser 的代码里，我们真实存在着三个互相调用的函数，它们严格对应着上一节讲的三大等级规则：

* **大当家：`parse_expression()`** (负责加减法)
* **二当家：`parse_term()`** (负责乘除法)
* **三当家：`parse_factor()`** (负责剥开纯数字或括号)

它们是怎么工作的？**永远是从最大的当家开始往下指派任务。这叫“下降（Descent）”。**

## 2. 核心源码逻辑再现（揭秘递归点）

我们看看这三段函数的心智模型长什么样：

```c
// ==========================================
// 【大当家】专门抓加减号
// ==========================================
AST_Node* parse_expression() {
    // 1. 大当家自己不管杂活，直接指派二当家去要一个乘除法块
    AST_Node* left = parse_term();  
    
    // 2. 然后大当家看看后面是不是跟着加号 (+) 或减号 (-)
    while (当前 Token 是 '+' 或者是 '-') {
        查收加号或减号;
        // 把加号右边的东西，再交给二当家去解析
        AST_Node* right = parse_term(); 
        
        // 3. 把二当家收上来的左右两堆东西，用加号节点（树根）拼成一棵小树！
        left = 制造一个加法节点(left, right);
    }
    return left;
}

// ==========================================
// 【二当家】专门抓乘除号
// ==========================================
AST_Node* parse_term() {
    // 1. 二当家也不管杂活，继续下降！指派三当家去拿最底层的数字
    AST_Node* left = parse_factor(); 
    
    // 2. 看后面是不是乘除号
    while (当前 Token 是 '*' 或者是 '/') {
        查收乘号或除号;
        AST_Node* right = parse_factor();
        left = 制造一个乘法节点(left, right);
    }
    return left;
}

// ==========================================
// 【三当家】真正的干活人（以及最恐怖的递归之门）
// ==========================================
AST_Node* parse_factor() {
    if (当前 Token 是纯数字) {
        // 第一种情况：终于到底了，抓取数字，做成树叶返回。
        return 制造一个数字节点(数值);
    } 
    else if (当前 Token 是 左括号 '(' ) {
        // 第二种情况：【高能预警】遇到括号了！
        吃掉这个 '(';
        
        // 【神奇的递归点】！！！
        // 括号里面是一整个大千世界，三当家处理不了，他干脆一通电话打给了大当家！！
        AST_Node* node = parse_expression(); 
        
        吃掉后面的 ')';
        return node;
    }
}
```

## 3. 为什么能无限套娃？（画境推演）

假设要解析的公式是：`2 * ( 3 + 4 )`

1. 程序启动！首先调用 **大当家 `parse_expression`**。
2. 大当家马上喊 **二当家 `parse_term`** 去拿左边的东西。
3. 二当家马上喊 **三当家 `parse_factor`** 去拿数字，三当家看到了 `2`，完美返回一棵数字节点树叶 `(2)`。
4. 此时，二当家往后看，看到了 `*` 号！他记录下 `*` 号，然后**再次呼叫**三当家去拿右边的东西。

**高潮来了：**
5. 三当家看到了接下来是 `(`。他心里一慌，遇到硬茬了。他吃掉 `(`，然后**重新拨通了大当家的电话 `parse_expression`！** 
6. 大当家接通电话，重新开始走流程，大当家指挥二当家，二当家指挥三当家，把括号里的 `3 + 4` 规规矩矩地解析出了一棵完美的小加法树。
7. 大当家把这棵小加法树交给三当家。
8. 三当家吃掉最后的 `)`，把这棵加法树返回给二当家。
9. 最终二当家把左边的 `(2)` 和右边的加法小树，用 `*` 号拼装在一起！成为了一棵完整的三维立体抽象语法树！

这就是**递归（Recursion）**的暴力粉碎美学！三当家在遇到括号绝境时，呼叫最高指挥官大当家重启整个流程。这个环一旦闭合，哪怕你写一万层括号，程序也绝对能把它严丝合缝地解析出来。
