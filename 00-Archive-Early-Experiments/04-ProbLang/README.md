# Probabilistic & AI-Native System Language (暂定名: ProbLang)

这是一门从零手写、直接生成 ARM64/X86 机器码（基于 LLVM 后端）的高级系统编程语言。

## 核心护城河（为什么它不是玩具？）
现有的语言（C/Java/Python）全都是建立在**冯·诺依曼架构的确定性逻辑**上的。它们的控制流只有单调的 `if/else`, `while`, `for`。它们的逻辑是绝对精确的。

但未来的计算是被 AI（模型）驱动的，充斥着**概率性（Probabilistic）和非确定性**。
`ProbLang` 在底层的语法树（AST）和汇编生成阶段，原生支持大模型与概率数学的特性。

### 独创语法一：原生分布抽样控制流
在传统的 C 语言里，你要写很复杂的 `rand()` 并判断。
在这里，概率是第一公民：
```scala
// 语言原生支持基于先验概率的分支预测
predictive_branch (0.7) {
    print("大概率执行这条时间线");
} diverge {
    print("小概率进入这个平行宇宙");
}
```

### 独创语法二：大模型推断作为底层运算符 (LLM as CPU ALU)
如果说传统的语言，`+` `-` `*` `/` 是 CPU 内部 ALU 模块提供的算术元语。
那么在 `ProbLang` 中，我们将 `LLM` 变成一个语言级的物理操作符 `~>`：
```scala
string review = "这个餐厅真的太难吃了，但我喜欢服务员的态度。";

// 操作符 ~> 并不是一个函数调用！
// 编译器会在底层通过 LLVM IR 直接将其编译为最极致优化的张量推理网络（或高并发 RPC 接口）
float sentiment_score = review ~> SentimentAnalysis; 

if (sentiment_score < 0.5) {
    trigger_alert();
}
```

## 工程目标（冲击 10万行代码）
1. **[0-1W 行] 前端编译器：** 手打极度健壮的 词法分析器(Lexer)、语法分析器(Parser)、类型推导系统(Type Checker)。
2. **[1W-3W 行] AST 设计：** 重新定义现代计算语言的抽象语法树，融合控制流图（CFG）与张量计算图。
3. **[3W-7W 行] LLVM 桥接引擎：** 将自定义语言的 AST 通过 C++ 翻译成 LLVM 中间表示代码（IR），利用苹果的优化通道生成变态级的高效机器码。
4. **[7W-10W 行] 自创协程与垃圾回收（GC）：** 我们不白嫖现成的运行时。徒手实现类似 Go 语言的 Goroutine 高并发微线程模型，以及写时拷贝的内存分配器。
