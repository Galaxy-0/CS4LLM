# 计算机科学基础学习路线图 (CS Fundamentals Roadmap)

这是一份关于CS基础学习路径的系统性规划，分为四个阶段，逐渐从“上层应用”向“底层硬件”深入，再横向扩展。

## 第一阶段：基石与逻辑
### 1. 编程语言基础
* **目标**：熟练掌握至少一门语言。建议从 C 或 C++ 开始，因为它们保留了指针和内存管理的概念，这对于后续理解操作系统和计算机组成至关重要。此外，可以再学一门高级语言（如 Python 或 Java）用于快速开发。
* **动作**：掌握基本语法、面向对象思想、内存的分配与释放。

### 2. 数据结构与算法（Data Structures & Algorithms）
* **重要性**：“程序 = 数据结构 + 算法”。这是所有编程的灵魂，不仅是日常写出高效代码的保证，更是所有大厂面试的绝对核心。
* **重点内容**：
  * 结构：数组、链表、栈、队列、哈希表、树（二叉树、红黑树、B树）、图。
  * 算法：排序、查找、递归、动态规划（DP）、贪心算法、深度/广度优先搜索（DFS/BFS）。
* **推荐资源**：《算法导论》或《算法》（作者 Sedgewick）；搭配力扣（LeetCode）刷题。

## 第二阶段：打通单机“任督二脉”
这一阶段的目的是理解代码是如何在计算机硬件上跑起来的。

### 3. 计算机组成原理（Computer Architecture）
* **重要性**：理解软硬件的交界处。写代码时考虑到程序的“局部性原则”以提升性能。
* **重点内容**：冯·诺依曼架构、指令集、CPU流水线、存储器层次结构（寄存器->L1/L2/L3 Cache->内存->硬盘）。
* **推荐资源**：《深入理解计算机系统》（CSAPP），强烈建议精读并完成配套 Lab 实验。

### 4. 操作系统（Operating Systems）
* **重要性**：底层硬件的“管家”。高级并发问题、性能调优本质上都是操作系统层面的问题。
* **重点内容**：
  * 进程与线程（调度与切换）。
  * 并发与同步（锁、信号量、死锁）。
  * 内存管理（虚拟内存、分页、分段）。
  * 文件系统与网络 I/O 模型（如 epoll/select）。
* **推荐资源**：《操作系统导论》（OSTEP）或《现代操作系统》；公开课首推 MIT 6.S081 (xv6)。

## 第三阶段：连接世界与工业运用
了解多台计算机如何通信，以及海量数据如何存储。

### 5. 计算机网络（Computer Networks）
* **重要性**：现代软件几乎全都是网络/分布式应用。
* **重点内容**：（自顶向下的学习方法）
  * 应用层：HTTP/HTTPS、DNS、WebSocket。
  * 传输层：TCP 的核心机制（三次握手/四次挥手、滑动窗口、拥塞控制）、UDP。
  * 网络层：IP协议、路由表分配。
* **推荐资源**：《计算机网络：自顶向下方法》；实践：Wireshark 抓包，或手写 HTTP Socket 服务器。

### 6. 数据库系统（Database Systems）
* **重要性**：业务系统的大脑。理解底层设计与优化。
* **重点内容**：关系代数、B+树索引结构、事务特性（ACID）、并发隔离级别及 MVCC 机制、锁机制。
* **推荐资源**：CMU 15-445 数据库公开课四大 Project。

## 第四阶段：屠龙技与化境 (进阶篇)

### 7. 编译原理（Compilers）
* **重要性**：带来极强的架构思维，对编程语言的设计哲学、代码优化、抽象语法树（AST）有深刻认知。
* **重点内容**：词法分析、语法分析、语义分析、中间代码生成、代码优化。
* **推荐资源**：《Crafting Interpreters》手把手教写一门脚本语言及虚拟机。

### 8. 分布式系统（Distributed Systems）
* **重要性**：应对高并发、高可用架构的关键。
* **重点内容**：CAP 定理、分布式共识算法（Paxos / Raft）、分布式锁、RPC。
* **推荐资源**：MIT 6.5840 (Distributed Systems) 公开课。
